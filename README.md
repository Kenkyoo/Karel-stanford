# 🤖 Karel the Robot - Stanford CS106A Solutions

Este repositorio contiene mis soluciones a los desafíos de **Karel el Robot**, utilizados en el curso de Ciencias de la Computación **CS106A de la Universidad de Stanford**.

Karel es una herramienta de aprendizaje basada en Python que permite desarrollar el pensamiento lógico y la resolución de problemas mediante un conjunto restringido de comandos.

## 🚀 El Desafío de Karel
A diferencia de la programación convencional, Karel no permite (en sus niveles iniciales) el uso de variables, aritmética o tipos de datos complejos. Esto me ha permitido fortalecer habilidades en:
- **Descomposición de problemas:** Dividir tareas complejas en funciones pequeñas y reutilizables.
- **Recursividad:** Resolver problemas de conteo y posicionamiento sin usar variables numéricas.
- **Control de Flujo:** Uso eficiente de estructuras `while`, `if/else` y precondiciones/postcondiciones.

## 📁 Proyectos Destacados

### 📍 Finding the Midpoint (Punto Medio)
**Problema:** Localizar el centro exacto de la primera fila en un mundo de cualquier dimensión ($N \times N$).
- **Lógica aplicada:** Implementé un algoritmo de "reducción de extremos" mediante el uso de beepers como marcadores físicos, permitiendo que Karel encuentre el centro sin conocer de antemano el ancho del mundo.

### 🏁 Checkerboard Karel
**Problema:** Crear un patrón de tablero de ajedrez con beepers en todo el mundo.
- **Lógica aplicada:** Manejo de condiciones de borde y cambio de filas, asegurando que el patrón sea consistente independientemente de si las dimensiones son pares o impares.

### 🧱 Steeplechase
**Problema:** Karel debe completar una carrera de obstáculos (paredes) de alturas variables.
- **Lógica aplicada:** Algoritmos de seguimiento de paredes y ascenso/descenso dinámico.

## 🛠️ Tecnologías
- **Lenguaje:** Python (Sintaxis de Karel)
- **Conceptos:** Clean Code, Modularización, Recursión.

## 📝 Notas de Aprendizaje
Durante este curso, aprendí que la eficiencia no siempre se trata de usar las funciones más avanzadas del lenguaje, sino de cómo estructurar la lógica para que el código sea robusto ante diferentes escenarios (mundos de distintos tamaños y formas).
